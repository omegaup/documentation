## Otimização de consultas e benchmarking de desempenho

**Breve descrição**:

Este projeto visa melhorar o desempenho da plataforma otimizando consultas SQL que atualmente precisam fazer varreduras completas de tabelas e implementando 2 testes de integração: 

* Um teste de integração que analisa o EXPLAIN de todas as consultas para determinar se é eficiente ou não.
* Um teste de integração que analisa o desempenho de todas as consultas executando-as em um conjunto de dados sintético com dimensões comparáveis ​​às de produção. 

O objetivo é garantir tempos de resposta rápidos para operações frequentes e garantir escalabilidade em cenários de alta carga. Mais detalhes sobre os requisitos podem ser encontrados em [este documento](https://docs.google.com/document/d/1X_fAm97L6_v9P8_R0S_Lp7X6e7x7j7z-X5z5-x5z5z5).

**Status atual**:

* Consultas lentas foram identificadas e documentadas em um [rastreador de consultas](https://docs.google.com/spreadsheets/d/1z5EZlGRY5MXUBYn5VoSX3c7Wqt14mI_iH-IY266bf_Y/edit?gid=0#gid=0).
* O progresso é acompanhado na edição mestre [#8277](https://github.com/omegaup/omegaup/issues/8277), que lista PRs criados e mesclados.
* Principais PRs: [#8450](https://github.com/omegaup/omegaup/pull/8450) (Script para preencher o banco de dados) e [#8423](https://github.com/omegaup/omegaup/pull/8423) (Script de detecção de consulta ineficiente).

**Resultados esperados**:

Redução significativa no tempo de execução de consultas já identificadas como ineficientes. Um sistema de benchmarking reproduzível para avaliar melhorias de desempenho. Os testes de integração evitam a introdução de novas consultas ineficientes no futuro.

**Habilidades preferidas**:

* Otimização de consulta SQL
* MySQL
*Píton
*PHP
*YML

**Possível mentor**:

[Ankitsinghsisodya](https://github.com/Ankitsinghsisodya), [pabo99](https://github.com/pabo99), [carlosabcs](https://github.com/carlosabcs)

## Migrando do Vue 2 para o Vue 3

**Breve descrição**:

## Migrando Bootstrap 4 para 5

## Integrar o Criador de Problemas

**Breve descrição**:

Um projeto do GSoC do ano passado introduziu o Problem Creator, um editor visual que ajuda os autores de problemas a criar e editar problemas com mais facilidade. No entanto, o Problem Creator ainda não está totalmente integrado ao omegaUp. Atualmente, os autores devem escrever seu problema no Creator, baixar um .zip e carregá-lo manualmente. Este projeto visa agilizar esses fluxos de trabalho integrando totalmente o Problem Creator com os recursos nativos de criação e edição do omegaUp.

**Status Atual**:

* Um [documento pdesign](https://docs.google.com/document/d/1qpBwJQ6QIiIXgWpb_qa6OJ8KpJPcd11x3qKiefdgPAw/edit?tab=t.0) técnico foi escrito para este projeto. 
* Alterações de back-end concluídas:
  * [#8470](https://github.com/omegaup/omegaup/pull/8470) Classes e validações de CDP, classes principais de CDP adicionadas e lógica de validação.
  * [#8479](https://github.com/omegaup/omegaup/pull/8479) Novo método no ProblemDeployer para modificar ZIPs problemáticos, permite atualizar o conteúdo ZIP problemático durante a implantação.
  * [#8554](https://github.com/omegaup/omegaup/pull/8554) Correções do CDPBuilder e cobertura de testes de unidade, problemas corrigidos no CdpBuilder e adição de um teste de unidade para garantir a correção.
  * [#8606](https://github.com/omegaup/omegaup/pull/8606) Suporte a idiomas no CDPBuilder, suporte multilíngue adicionado com prioridade baseada na preferência de idioma
  * [#8613](https://github.com/omegaup/omegaup/pull/8613) Lógica de back-end para edição de casos e CDP, lógica do lado do servidor adicionada para suportar edição de casos e atualizações de CDP.
  * Alteração inicial da IU concluída: [#8593](https://github.com/omegaup/omegaup/pull/8593) Os componentes CasesForm e DeleteConfirmationForm introduziram os formulários de IU básicos necessários para gerenciar casos
* Trabalho pendente (principalmente relacionado à UI)
  * [#8471](https://github.com/omegaup/omegaup/pull/8471) Integre o CDP ao formulário do problema e lide com ZIPs externos
  * [#8492](https://github.com/omegaup/omegaup/pull/8492) Implementação de UI para a interface de edição
  * [#8595](https://github.com/omegaup/omegaup/pull/8595) Integração de UI: visualização de edição, CaseEdit e barra lateral

**Resultados esperados**:

Uma experiência de usuário completa e perfeita, onde os autores podem criar ou editar problemas diretamente por meio do Problem Creator, sem transferências manuais de arquivos.

# Como acelerar

Para se familiarizar com [omegaUp.com](https://omegaup.com) e começar a colaborar conosco, siga estas etapas:

 - Leia [este artigo](http://www.ioinformatics.org/oi/pdf/v8_2014_169_178.pdf) publicado por nossos cofundadores para aprender sobre a arquitetura e o design geral de nossa plataforma.
 - Leia [este artigo de acompanhamento](https://ioinformatics.org/journal/v18_2024_167_174_Duenas.pdf) para saber mais sobre o trabalho realizado nos anos mais recentes no omegaUp.
 - Leia nossa [documentação técnica](../../index.md) neste site (e o [repositório omegaUp](https://github.com/omegaup/omegaup) para código-fonte).

# Processo de inscrição

#### Nosso processo de inscrição consiste em três fases. Se você quiser participar conosco este ano, deverá preencher cada uma delas na ordem.

### Fase Um: Conclua nosso teste 

 - Primeiro, crie uma conta em [omegaUp.com](https://omegaUp.com).
 - Participe do nosso GSoC 2026 [teste omegaUp](https://omegaup.com/contest/gsoc2026). O teste consiste em 3 problemas, você tem que resolver pelo menos 2 deles para passar. **No caso de plágio, desqualificaremos os candidatos envolvidos**, portanto, não compartilhe suas soluções com outros candidatos.
#### nota: url alternativo para o teste: [omegaUp Test](https://omegaup.com/arena/gsoc2026)

### Fase Dois: Familiarize-se com nossa base de código

Pedimos que você conclua a primeira fase antes de começar a trabalhar com nossa base de código.- Siga o guia [configuração do ambiente de desenvolvimento](../../getting-started/development-setup.md).
 - Encontre um bug interessante para resolver em nosso [rastreador de problemas](https://github.com/omegaup/omegaup/issues) (especialmente em nossa lista de ["Bom primeiro problema"](https://github.com/omegaup/omegaup/labels/Good%20first%20issue), ou entre em contato com o [canal Discord](https://discord.gg/gMEMX7Mrwe) solicitando um e teremos o maior prazer em encontrar uma boa opção para você. A maioria das conversas no rastreador de problemas são em espanhol, mas sinta-se à vontade para mudar a conversa para inglês sobre qualquer problema. Alternativamente, você pode ir e encontre bugs no omegaup.com, relate-os em nosso rastreador de problemas e corrija-os.
 - Implemente sua correção e envie-a para revisão. Depois de mesclado, você pode passar para a terceira fase.

### Fase Três: Escrevendo sua proposta

Nesta etapa esperamos que você esteja familiarizado com nosso ambiente de desenvolvimento e código, pois isso facilita a compreensão de nossas ideias de projeto. **Pedimos que você mescle pelo menos três PR em um dos repositórios omegaUp antes de trabalhar no design de um projeto específico.**

 - Crie um documento de design para o seu projeto usando [este modelo](https://docs.google.com/document/d/1_FKfpc2M3VLDVYqvT8ZgsgwIJ3zaZnyUVmSm-H3h6UQ/edit). Se quiser trabalhar em mais de um projeto, pedimos que mencione isso na sua candidatura, mas inclua apenas um design. Isso reduz a carga de trabalho dos revisores. 
 - Também encorajamos você a **enviar-nos seu rascunho de proposta para revisar e dar feedback**. Envie o link do seu rascunho através deste formulário `https://forms.gle/TbbscnWA5B2ZWfJq7`. Certifique-se de que qualquer pessoa com o link possa ver e comentar.
 - Tentaremos fornecer o máximo de feedback possível e o mais rápido possível. No entanto, não forneceremos feedback aos candidatos que não tenham concluído com êxito as fases 1 ou 2.
 - Quando você considerar que sua inscrição está pronta, não esqueça de **enviá-la para [Google](https://summerofcode.withgoogle.com/age-verification/student/?next=%2Fstudent-signup%2F)** porque se não fizer isso, você não poderá ser considerado no GSoC 2026.

### Fase Quatro: Entrevista com a organização
Após o envio dos documentos de design, selecionaremos uma pequena lista de candidatos com base nas 3 primeiras fases e agendaremos entrevistas por telefone com eles. A entrevista consistirá em questões comportamentais e técnicas.

Consideraremos apenas candidatos que concluíram todas as 4 fases da inscrição.

# Perguntas frequentes #
   * **O script de instalação do ambiente de desenvolvimento está gerando um erro.** Siga nosso guia [obtenção de ajuda](../../getting-started/getting-help.md) para resolver seu problema.
   * **Devo falar espanhol?** Claro que não. Tentamos o nosso melhor para sermos o mais inclusivos possível para os voluntários não espanhóis. Fique à vontade para usar o inglês em todos os nossos canais de comunicação e em seu código. Também descobrimos que o Google Translate faz um trabalho decente na tradução do conteúdo em espanhol de nossa página GitHub. Recomendamos que você o use para navegar em nosso rastreador de problemas, wiki, etc.
  * **Quantas vagas sua organização terá para o GSoC 2026?** Pediremos 4 alunos este ano, mas ainda não há garantia, teremos certeza até meados de maio de 2026.
  * **Como você escolhe seus alunos?** Analisaremos cada inscrição que recebermos e escolheremos nossos candidatos com base em três coisas:
    * Nível de habilidade do candidato. Existem duas boas maneiras de mostrar seu nível de habilidade em sua aplicação: através de pull requests impactantes enviados para nossos repositórios (esta é a forma recomendada), ou através de experiência anterior. Certifique-se de incluir evidências de pelo menos um deles em sua inscrição.
    * Plano de trabalho do candidato. Pedimos que você escreva um design de alto nível do seu projeto seguindo nosso [modelo de proposta](https://docs.google.com/document/d/1_FKfpc2M3VLDVYqvT8ZgsgwIJ3zaZnyUVmSm-H3h6UQ/edit)
    * Ajuste cultural. Gostamos de pessoas que promovem a inclusão na organização e ajudam proativamente os colegas. Uma boa maneira de ajudar outros candidatos quando eles fazem perguntas no [canal Discord](https://discord.gg/gMEMX7Mrwe).
